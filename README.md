# Pantrypoints

A moneyless economy webapp built with SvelteKit 5, Tailwind CSS v4, Paraglide JS (i18n), Drizzle ORM + Turso, and deployed to Cloudflare Pages.

## Tech Stack

| Layer | Tool |
|---|---|
| Framework | SvelteKit 2 + Svelte 5 |
| Styling | Tailwind CSS v4 + Typography + Forms |
| i18n | **Paraglide JS v2** (`@inlang/paraglide-js`) |
| Database | Turso (libSQL) + Drizzle ORM |
| Markdown | mdsvex |
| Icons | lucide-svelte |
| Deployment | Cloudflare Pages (`@sveltejs/adapter-cloudflare`) |
| Package mgr | Bun |

## i18n with Paraglide

Translations live in flat JSON files:

```
messages/
  en.json   ← source language
  zh.json   ← Chinese
```

The Vite plugin compiles these into `src/lib/paraglide/` (auto-generated, not committed).

In any `.svelte` file:

```svelte
<script>
  import * as m from '$lib/paraglide/messages';
  import { languageTag } from '$lib/paraglide/runtime';
</script>

<h1>{m.hero_title()}</h1>
<p>{languageTag() === 'zh' ? app.nameZh : app.name}</p>
```

Language is set per-request in `src/hooks.server.ts` from the `lang` cookie and passed down via `event.locals.lang`. The layout calls `setLanguageTag(data.lang)` on the client to keep it in sync.

**To add a new language:**
1. Add the tag to `project.inlang/settings.json` → `languageTags`
2. Create `messages/<tag>.json` with all keys
3. Add a button in `Nav.svelte`

**To add a new message key:**
1. Add it to `messages/en.json` with the English value
2. Add the same key to `messages/zh.json` with the Chinese value
3. Use `m.your_key()` in any component — TypeScript will autocomplete

## Quick Start

```bash
# Install
bun install

# Copy env
cp .env.example .env
# Fill in TURSO_URL, TURSO_AUTH_TOKEN, ADMIN_PASSWORD

# Push DB schema to Turso
bun run db:push

# Start dev (Paraglide compiles automatically)
bun run dev
```

## Turso Setup

```bash
# Install Turso CLI
curl -sSfL https://get.tur.so/install.sh | bash

# Create database
turso db create pantrypoints

# Get URL and token
turso db show pantrypoints --url
turso db tokens create pantrypoints

# Run migration
turso db shell pantrypoints < migrations/0001_initial.sql
```

## Deployment to Cloudflare Pages

```bash
bun run build
bunx wrangler pages deploy .svelte-kit/cloudflare
```

Set these environment variables in the Cloudflare dashboard under **Settings → Environment Variables**:
- `TURSO_URL`
- `TURSO_AUTH_TOKEN`
- `ADMIN_PASSWORD`

│   │       ├── schema.ts

│   ├── routes/
│   │   ├── +layout.server.ts   ← Passes lang from locals to client
│   │   ├── +layout.svelte      ← setLanguageTag() on client
│   │   ├── +page.svelte        ← Landing page + registration
│   │   ├── apps/               ← App listing + [slug] showcase
│   │   ├── news/               ← News listing + [slug] reader

└── migrations/

## Admin Access

Visit `/admin`. Default dev password: `admin123` (set `ADMIN_PASSWORD` in prod).

## Adding Content

**News article:**
Drop a `.md` file in `src/content/en/` and `src/content/zh/` with frontmatter:
```markdown
---
title: Your Title
description: Short description shown in listings
date: 2025-06-01
author: Author Name
tags: [tag1, tag2]
---
Content...
```

**New app:**
Add an entry to the `apps` array in `src/lib/apps.ts`. The `/apps/[slug]` showcase page is automatic.
